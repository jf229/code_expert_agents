Each agent implements a different retrieval strategy:
• Top-K: Fast similarity search (k=5 documents)
• Iterate: Process all files, synthesize comprehensive view
• Graph: NetworkX knowledge graph with entity relationships
• Multi-Rep: Adaptive strategy selection based on question type
Key Features
• Multi-provider LLM support (Ollama, WCA, OpenAI, Claude)
• Local embeddings for privacy (Ollama)
• Intelligent document filtering & repository analysis
• Unified response generation with source attribution
• Central configuration management
• Extensible plugin architecture
Best Use Cases
Top-K: Quick specific questions
Iterate: Full project overview
Graph: Entity relationships & dependencies
Multi-Rep: Adaptive analysis for varied question types

All agents share:
• Central configuration (config.yaml)
• Shared services and infrastructure
• Consistent response formatting
